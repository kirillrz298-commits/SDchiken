# SD Chicken Website 🍗

Современный и адаптивный веб-сайт для ресторана быстрого питания **SD Chicken**. Сайт полностью подготовлен к публикации и локальному запуску.

## 🚀 Описание проекта
Сайт состоит из главного лендинга и интерактивной страницы меню с поддержкой корзины заказов, сохранения отзывов через LocalStorage, и секции FAQ. Весь дизайн переведён на премиальную темно-серую цветовую гамму, а изображения оптимизированы для корректного отображения в веб-среде.

---

## 🛠️ Локальный запуск проекта

Проект включает в себя простой Node.js Express сервер для локальной разработки и тестирования.

1. Установите зависимости (если еще не установлено):
   ```bash
   npm install
   ```

2. Запустите локальный сервер:
   ```bash
   npm start
   ```
   или
   ```bash
   npm run dev
   ```

3. Откройте в браузере: [http://localhost:3000](http://localhost:3000)

## ☁️ Инструкции по публикации (Деплой на Netlify)

Данный проект полностью готов к публикации на хостинг **Netlify**. Вы можете развернуть его двумя способами.

### Способ 1. Автоматический деплой через GitHub (Рекомендуемый)
Этот способ настраивает автоматическое обновление сайта при каждом вашем `git push`.

1. Загрузите проект на ваш GitHub-репозиторий (инструкции см. ниже).
2. Зарегистрируйтесь/войдите в аккаунт на [Netlify](https://www.netlify.com/).
3. На панели управления Netlify нажмите **"Add new site"** ➔ **"Import an existing project"**.
4. Выберите **GitHub** и авторизуйтесь.
5. Выберите репозиторий `SDchiken`.
6. Нажмите **"Deploy SDchiken"** (настройки сборки не требуются, так как в проекте присутствует готовый конфигурационный файл `netlify.toml`).
7. Готово! Теперь при изменении кода и отправке его на GitHub, сайт на Netlify обновится автоматически.

### Способ 2. Ручной деплой (Drag & Drop)
Самый простой и быстрый способ без использования Git.

1. Перейдите на страницу [Netlify Drop](https://app.netlify.com/drop).
2. Перетащите папку проекта со своего компьютера в зону загрузки на экране.
3. Проект будет опубликован в течение нескольких секунд. Вы получите ссылку на сайт.

### Способ 3. Ручной деплой через Netlify CLI
1. Установите CLI глобально:
   ```bash
   npm install -g netlify-cli
   ```
2. Авторизуйтесь:
   ```bash
   netlify login
   ```
3. Выполните команду инициализации и деплоя:
   ```bash
   netlify deploy --prod
   ```

## 📁 Настройка Git и отправка на GitHub

Репозиторий Git уже инициализирован локально, и remote-адрес настроен на ваш репозиторий `https://github.com/kirillrz298-commits/SDchiken.git`.

Для отправки изменений на GitHub используйте следующие команды:

1. Переключитесь на ветку `main` (или `master`):
   ```bash
   git branch -M main
   ```

2. Отправьте изменения на удаленный репозиторий:
   ```bash
   git push -u origin main
   ```
   *(При первом выполнении команды Git может попросить вас пройти авторизацию в GitHub через браузер или Token).*

Если в будущем вы внесете новые изменения, используйте стандартный рабочий процесс:
```bash
git add .
git commit -m "Описание изменений"
git push
```

## 🎨 Выполненные изменения дизайна и структуры
1. **Премиальный темный фон**: Фоновый цвет `body` изменен со стандартного серого `#2b2b2b` на стильный глубокий темно-серый `#121212`.
2. **Очистка фона**: Удалены отвлекающие или конфликтующие стили на секциях (`.advantages`, `.promos`, `.faq`, `.contacts`, и т.д.), чтобы они идеально сочетались с новой темой.
3. **Безопасные имена изображений**: Файлы изображений в папке меню переименованы из Cyrillic (русские символы и пробелы) в безопасные ASCII имена (`menu1/sd-burger.png`, `menu1/classic-burger.png` и т.д.), что исключает ошибки 404 на Linux-серверах хостингов.
4. **Конфигурационные файлы**: Созданы файлы `.gitignore`, `tsconfig.json`, `package.json`, `server.js` и `netlify.toml`.
